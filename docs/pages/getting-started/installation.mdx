# Installation

The easiest way is the **standalone binary** — one small download per OS, no Python
required. Developers can also install from source.

import { Callout } from 'nextra/components'

## Download the binary (recommended)

Each release ships a tiny native launcher for every OS. On its **first run** it
provisions a private Python runtime and installs PyTorch with the right backend for your
machine — CPU, NVIDIA CUDA, or Apple MPS — **automatically**. Nothing else to install.

| OS | Download |
|---|---|
| macOS (Apple Silicon) | [`eujeno-macos-arm64`](https://github.com/babelfornet/eujeno/releases/latest/download/eujeno-macos-arm64) |
| macOS (Intel) | [`eujeno-macos-x64`](https://github.com/babelfornet/eujeno/releases/latest/download/eujeno-macos-x64) |
| Linux (x86-64) | [`eujeno-linux-x64`](https://github.com/babelfornet/eujeno/releases/latest/download/eujeno-linux-x64) |
| Linux (ARM64) | [`eujeno-linux-arm64`](https://github.com/babelfornet/eujeno/releases/latest/download/eujeno-linux-arm64) |
| Windows (x86-64) | [`eujeno-windows-x64.exe`](https://github.com/babelfornet/eujeno/releases/latest/download/eujeno-windows-x64.exe) |

**macOS / Linux:**

```bash
curl -fsSL https://github.com/babelfornet/eujeno/releases/latest/download/eujeno-macos-arm64 -o eujeno
chmod +x eujeno
./eujeno --help
```

**Windows (PowerShell):**

```powershell
irm https://github.com/babelfornet/eujeno/releases/latest/download/eujeno-windows-x64.exe -OutFile eujeno.exe
.\eujeno.exe --help
```

Then **join an existing network** or **create a new one**:

```bash
# join a network that already serves a model
./eujeno serve --peers http://SEED_IP:8001 --model Qwen/Qwen2.5-7B-Instruct

# …or create a new single-box network for a model
./eujeno up --model Qwen/Qwen2.5-7B-Instruct
```

<Callout type="info">
  The first run downloads a few hundred MB (managed Python + PyTorch + eujeno) into
  `~/.eujeno/runtime` (`%LOCALAPPDATA%\eujeno` on Windows); later runs start instantly.
  On macOS you may need to clear the quarantine flag once:
  `xattr -d com.apple.quarantine ./eujeno`.
</Callout>

## From source (for development)

To hack on Eujeno, clone the repo and use the auto-bootstrapping `bin/eujeno` launcher.

### Prerequisites

- **Python 3.10+** and the `python -m venv` module.
- **git**, to clone the repository.
- Enough RAM/disk for the model you intend to run. Models are downloaded from
  [Hugging Face](https://huggingface.co) on first use, and each node only loads the
  layers it hosts — so a single node can cover a small model, or several nodes can
  share a larger one.

### Clone the repository

```bash
git clone <repository-url> eujeno
cd eujeno
```

### Bootstrap with the launcher (recommended)

The `bin/eujeno` launcher is auto-bootstrapping: the first time you run it, it creates
a `.venv` and installs the package (`pip install -e .`), then forwards your command. No
manual environment setup needed.

```bash
./bin/eujeno --help                 # first run: creates .venv + pip install -e . , then runs
```

Every later invocation reuses the same `.venv` and runs immediately. You can keep using
`./bin/eujeno <command>` for everything.

### Manual installation (alternative)

If you prefer to manage the virtual environment yourself:

```bash
python -m venv .venv && . .venv/bin/activate && pip install -e .
eujeno --help
```

With the environment activated, the `eujeno` command is on your `PATH` directly (no
`./bin/` prefix).

## Verify the install

Either path should leave you with a working CLI. Confirm it:

```bash
eujeno --help
```

You should see the list of commands (`up`, `serve`, `infer`, `coordinator`, `models`,
and more). To list the models Eujeno can run:

```bash
eujeno models
```

<Callout type="info">
  Throughout the docs we write commands as `eujeno <command>`. If you used the launcher
  without activating the venv, run them as `./bin/eujeno <command>` instead — they are
  equivalent.
</Callout>

## Updating to a new version

There is no `eujeno update` command — you just **replace the launcher binary**, and the
next time you run a real command the launcher notices its version changed and **re-provisions
the matching runtime automatically** (it re-installs the new wheel on first run; later runs
start instantly).

**Binary (macOS / Linux)** — re-run the one-line installer, which fetches the latest release:

```bash
curl -fsSL https://eujeno.com/install.sh | sh
```

**Binary (Windows)** — download the latest `eujeno-windows-x64.exe` again from the
[releases page](https://github.com/babelfornet/eujeno/releases/latest) and replace your copy.

<Callout type="info">
  Pin a specific version instead of the latest with `EUJENO_VERSION=v0.1.4 curl -fsSL
  https://eujeno.com/install.sh | sh`. If a run ever seems stuck on the old runtime, force a
  clean re-provision with `EUJENO_FORCE_BOOTSTRAP=1 eujeno --help`.
</Callout>

**From source** — pull the latest code; the editable install reflects code changes
immediately. Re-run `pip install -e .` only if a release changed the **dependencies** (the
`bin/eujeno` launcher reinstalls only when the package is missing, not on dependency bumps):

```bash
git pull
.venv/bin/pip install -e .     # only needed if dependencies changed
./bin/eujeno --version
```

You can always check what you're running with `eujeno --version`.
