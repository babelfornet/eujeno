# Join the Swarm

The fastest way to contribute a node to an existing network is **self-assignment**.
Instead of picking which layers to host by hand, your node reads the current coverage
gaps from a seed peer and its own RAM budget, then **claims** a contiguous range that
fits. As more nodes join, the model assembles itself across the swarm.

import { Callout } from 'nextra/components'

## One command

```bash
eujeno serve --auto --peers http://192.168.1.10:8001 \
  --model Qwen/Qwen2.5-7B-Instruct --dtype bfloat16
```

This connects to the seed at `http://192.168.1.10:8001`, inspects which layers are
already covered, measures how many layers fit in the available RAM, and serves the
range needed to fill a gap.

In **coordinator mode** `--auto` works the same way — point it at the coordinator and it
reads coverage from the coordinator's own `/registry` (no separate `--peers` needed):

```bash
eujeno serve --auto --coordinator ws://COORDINATOR:9000/node \
  --model Qwen/Qwen2.5-7B-Instruct --dtype bfloat16
```

<Callout type="info">
  The model becomes **operational** only once coverage is complete: `embed` + all
  contiguous decoder ranges + `head`. Until then, `infer` responds `NOT_OPERATIONAL`.
  Each `--auto` node claims part of the missing range, so the network assembles
  progressively as nodes join.
</Callout>

## The flags

| Flag | What it does |
|---|---|
| `--auto` | Auto-assign layers from registry gaps + RAM capacity. The node plans its own range at startup instead of using `--stages`. |
| `--peers <seed>` | Comma-separated seed peers for gossip discovery. From a seed the node learns the current topology and coverage. The first node in a swarm needs no seed; every other node lists at least one. |
| `--coordinator <ws>` | Coordinator WS URL. In coordinator mode `--auto` reads coverage from the coordinator's `/registry` automatically — you don't also need `--peers`. |
| `--ram <GB>` | Forces the memory budget used for planning. Defaults to the node's detected free RAM. |
| `--reserve <fraction>` | Fraction of RAM kept aside for activations / KV-cache / OS. Default `0.2`. |
| `--target <N>` | Desired replicas per range. `--target 2` aims for **≥2 nodes per range** for redundancy. Default `1`. |
| `--dtype <type>` | `float32`, `bfloat16`, or `float16`. Default: **bfloat16 on GPU, float32 on CPU**. `bfloat16` halves the RAM and lets the node hold more layers. |
| `--model <id>` | Hugging Face model ID. Must match the model the rest of the swarm is serving. |

## Startup order matters

Claiming is a **one-time decision made at startup**. Start the seed(s) first and wait
until they are reachable and have published their range **before** launching other
`--auto` nodes.

<Callout type="warning">
  If a node plans while the seed's registry is still empty, it sees no existing coverage
  and may claim too much. The node **retries the coverage read** to ride out a transient
  network blip, but it can't wait for a seed that hasn't published yet — start and warm up
  the seed first (confirm `SEED/registry` is non-empty). For a guaranteed split, use
  explicit [stages](/running-a-node/layers-and-ram) instead, or stagger node startup.
</Callout>

## Redundancy with `--target`

Pass `--target 2` so the planner aims for at least two nodes covering every range.
When a range has two holders and one goes down mid-job, the network can continue on
the other. The node also announces its `capacity` in the gossip record, which lets
later nodes plan around it.

```bash
eujeno serve --auto --target 2 --peers http://192.168.1.10:8001 \
  --model Qwen/Qwen2.5-7B-Instruct --dtype bfloat16
```
